This document describes APIs for the com.samsung.android.sdk.healthdata of Samsung Digital Health SDK.
All S Health's partner applications need to synchronize health data with the Health Data package of Samsung Digital Health SDK.

Health Data Package

The packages above contain classes and interfaces which enable you to create applications with the following functionalities.

Health Data Store

Samsung Digital Health applications can save its health data to S Health or read S Health's data through the health data store connection.
See Health Data Store for more information.

Health Data Type

Samsung Digital Health provides useful predefined and custom data types.
See Health Data Types for more information.

Privacy

Data permission acquisition is required to access a specific data type.
See Permission Manager for more information.

For more information related to the development environment, fundamentals, features and sample descriptions, see Programming Guide. You can find it in the downloaded SDK's /HealthData/Docs folder.

Compatible version of S Health

Samsung Digital Health SDK works with S Health 4.0 or above.
And some APIs of the Health Data package cannot work with the specified version of S Health.
See the following table for more information.

Heath Data package version Compatible version of S Health
1.0.0
1.1.0		 4.0 or above
1.2.0 or above 4.6 or above

E.g. if you create your application with the Health Data package 1.2.0, it requires to install S Health 4.6 or above in the device.

The following guide will help you to prevent the related error regardless of whether you use any version of the Health Data package.

  1. Connect to the health data store with HealthDataStore.connectService()
  2. Check its error with HealthConnectionErrorResult.onConnectionFailed().
  3. If the error is OLD_VERSION_PLATFORM,
    it means that your application works with the latest S Health version.
  4. Check if Samsung Digital Health provides a resolution for the error with HealthConnectionErrorResult.hasResolution().
  5. If yes, resolve the error with HealthConnectionErrorResult.resolve().
    The platform starts an activity to install the latest S Health on the app market.
